# 🎛️ Spectrogram Studio – Zero-Dependency DSP Workspace

A browser-based **Digital Signal Processing (DSP)** application that demonstrates **2D Discrete Fourier Transform (DFT)**, **frequency-domain filtering**, and **Inverse DFT (IDFT)** using **pure JavaScript** with **no external libraries**.

Users can load an image, inspect its frequency spectrum, manually suppress unwanted frequency components, and reconstruct the denoised image in real time.

---

## ✨ Features

* 📁 Load any grayscale or color image
* ⚡ Pure JavaScript implementation of **2D Discrete Fourier Transform**
* 📊 Log-compressed, center-shifted frequency spectrum visualization
* 🎨 Interactive frequency masking with adjustable brush size
* 🔄 Manual **Inverse DFT** reconstruction
* 📈 Performance telemetry

  * Forward transform latency
  * Inverse transform latency
  * Estimated Signal-to-Noise Ratio (SNR)
* 💻 Terminal-style processing log
* 🚫 No external libraries or frameworks
* 🌐 Runs entirely inside the browser

---

## 🖥️ Project Preview

The workspace consists of three processing stages:

1. **Input Spatial Domain**

   * Displays the uploaded image
   * Converts it to grayscale for processing

2. **Frequency Domain**

   * Visualizes the Fourier spectrum
   * Supports interactive masking to suppress periodic noise

3. **Output Spatial Domain**

   * Displays the reconstructed image after applying the Inverse DFT

---

## 🧠 How It Works

### Step 1 – Load an Image

Upload an image using the **Load Target Image** button.

Large images are automatically resized (maximum 40×40 pixels) to keep the computational complexity manageable since the implementation uses the standard **O(N⁴)** nested-loop DFT algorithm.

---

### Step 2 – Forward DFT

The application computes the **2D Discrete Fourier Transform** using the mathematical definition:

[
F(u,v)=\sum_x \sum_y f(x,y)e^{-j2\pi(\frac{ux}{W}+\frac{vy}{H})}
]

The transform separates the image into:

* Real coefficients
* Imaginary coefficients

which together represent the frequency-domain information.

---

### Step 3 – Frequency Spectrum

The magnitude spectrum is generated by:

* Computing frequency magnitudes
* Performing FFT-style quadrant shifting
* Applying logarithmic compression
* Rendering the normalized spectrum onto the canvas

This makes dominant frequency spikes easier to identify.

---

### Step 4 – Interactive Frequency Masking

Use the adjustable brush to paint directly over unwanted frequency components.

Painting:

* Sets selected frequency coefficients to zero
* Removes periodic frequency spikes
* Updates the spectrum interactively

---

### Step 5 – Inverse DFT

The application reconstructs the image using the mathematical inverse transform:

[
f(x,y)=\frac1{WH}\sum_u\sum_vF(u,v)e^{j2\pi(\frac{ux}{W}+\frac{vy}{H})}
]

The reconstructed output is displayed in the Output canvas.

---

## 📊 Performance Metrics

The application displays:

* Forward DFT execution time
* Inverse DFT execution time
* Estimated Signal-to-Noise Ratio (SNR)
* Processing status logs

These metrics help visualize the computational cost of the transform.

## ⚠️ Limitations

* Implements the mathematical DFT directly instead of FFT.
* Computational complexity is **O(N⁴)**.
* Images larger than 48×48 pixels are automatically resized to 40×40.
* Intended for educational and visualization purposes rather than high-performance image processing.
